# 多节点 mTLS 控制面与节点部署

## Goal

将当前单节点 NativeS3-Bridge 演进为一个国内集中面板管理多个海外节点的部署形态。海外节点继续直接承载 S3 数据流量，只通过主动发起的 mTLS 管理连接接受面板管理，不在节点上暴露任何管理相关监听端口。

## Confirmed Facts

- 采用一个代码仓库、两个程序入口、两个容器镜像：面板镜像和节点镜像。
- 面板部署在国内机器，节点部署在海外机器。
- 节点主动连接面板；面板不反向连接节点。
- 管理通道从第一版开始使用 mTLS，每个节点具有独立身份，能够单独吊销。
- 节点仍正常监听 S3 服务端口，S3 请求和响应直接经过节点自身网络出口，不经过国内面板。
- 节点不得监听 Web 管理、管理 API、远程控制等管理相关端口。
- 当前仓库只有一个 `natives3bridge` 入口；进程会同时启动 S3 服务和 WebAdmin 服务。
- 当前 WebAdmin、S3 凭据、Bucket、配额、请求统计和部分运行配置依赖节点本地数据库。
- 当前 WebAdmin 前端嵌入 Go 二进制，现有 Docker 镜像同时暴露 9000 和 9001。
- S3 凭据、Bucket、配额、运行数据和对象命名空间均为节点级独立资源，不构成跨节点共享的全局资源。
- 面板提供集中管理、状态汇总和面向多个节点的批量操作，但不承诺各节点对象数据或资源状态天然一致。
- 面板是 S3 凭据、Bucket、配额、Webhook、限流策略等运行期业务配置的唯一管理权威。
- 节点本地配置仅负责启动必需的基础设施参数，例如面板地址、节点证书路径、S3 监听地址、数据目录、数据库连接和 S3 TLS。
- 面板失联时，节点继续使用最后一次成功应用并持久化的业务配置，不允许通过节点管理接口产生新的业务配置变更。
- 新节点由管理员先在面板创建，并获取高强度、单次使用、短时有效的注册令牌；不设置额外的二次人工审批步骤。
- 节点首次启动时在本机生成私钥，私钥不得上传到面板；节点提交注册令牌和证书申请，由面板签发独立客户端证书。
- 注册令牌成功使用后立即失效；完成注册后，所有常规管理连接必须使用 mTLS，不允许继续使用注册令牌认证。
- 第一版面板以现有 WebAdmin 的节点级能力为基础，新增节点注册、连接状态和证书生命周期管理。
- 第一版支持按节点查看仪表盘、运行状态、有限日志、配置版本和状态漂移，并管理凭据、Bucket、ACL、配额及存储扫描/校正。
- 第一版不支持远程 Shell、节点程序升级、服务或容器重启、操作系统管理、对象复制、自动故障转移和大规模实时日志汇聚。
- 第一版面板采用单实例部署，不实现主备切换或多实例 Active-Active。
- 面板不可用期间仅暂停管理功能；节点必须继续运行 S3 服务，并在面板恢复后自动重连和对账。
- 小规模部署允许使用 SQLite；正式生产推荐 PostgreSQL 或 MySQL，并要求备份面板数据库、CA 数据和审计记录。
- 第一版延续现有单管理员密码与 Session 认证，保留登录失败锁定、TOTP 和可选验证码能力。
- 第一版不引入多用户、邀请、角色或节点级 RBAC；所有管理操作仍必须写入审计日志并记录统一管理员身份。
- 面板必须加密保存 S3 Secret Key，以便节点重建、恢复和状态对账；用于加密的主密钥必须与数据库分离保存。
- S3 Secret Key 只在创建时向管理员完整展示一次，之后不得通过列表、详情、日志或审计接口再次返回；需要更换时执行凭据轮换。
- 已有单节点部署必须支持原地接入：保留对象目录和节点数据库，完成 mTLS 注册后只读上报现有 Bucket、凭据、配额和当前状态。
- 面板必须先展示导入摘要并由管理员确认；导入完成前不得向节点覆盖业务配置，导入成功后生成初始配置基线，此后切换为面板权威。
- 迁移过程不得上传或搬迁对象文件，也不迁移历史日志；现有 Secret Key 仅可通过已建立的 mTLS 通道导入并在面板加密保存。
- 节点离线时允许修改业务配置；面板只保存最新期望状态和配置版本，不逐条积压已被覆盖的配置命令。
- 节点重连后必须直接对账并应用最新期望版本，页面应明确显示等待同步、已同步、失败和状态漂移。
- 日志查询、存储扫描、存储校正等一次性任务只允许向在线节点发起，节点离线时不得排队等待未来自动执行。
- 一次性任务发出后连接中断时必须标记为失败或结果未知，并要求管理员重新确认，不得静默重试高风险操作。
- 节点生命周期包含 `active`、`disabled` 和 `retired`：停用可恢复，退役永久生效并吊销全部证书与注册令牌。
- 节点重装或替换必须保留原逻辑节点、期望配置和历史归属，同时吊销旧证书并通过新的单次注册令牌签发新证书。
- 节点内部唯一标识不得复用；节点显示名称可以修改。UI 中的删除操作必须实现为保留审计关系的永久退役，而非物理删除。
- 灾难恢复采用失败关闭：完整备份应支持原节点自动重连；CA 或 Secret Key 加密关键密钥确实丢失时，接受重新注册节点或轮换 S3 凭据，不提供绕过 mTLS 或导出明文密钥的恢复后门。
- 内部 PKI 使用离线根 CA 与在线中间 CA；面板日常节点证书签发和续期只使用在线中间 CA。
- 本任务目前只进行规划，不开始代码实现，也不激活执行阶段。

## Requirements

- 面板必须提供统一的节点注册、在线状态、配置管理、任务下发、结果查看和审计入口。
- 面板必须管理节点 mTLS 身份的注册、签发、续期和吊销生命周期。
- 首次注册请求必须通过正常的服务器 TLS 校验面板身份，注册令牌默认有效期规划为 10 分钟，并支持管理员提前作废。
- 节点 Agent 必须支持主动连接、心跳、断线重连、身份校验、配置同步、任务接收和执行结果回传。
- 单个节点身份泄漏或节点下线时，必须能够只禁用该节点，不影响其他节点。
- 面板不可用或跨境网络中断时，不得中断节点已有的 S3 数据面服务。
- 节点必须持久化最后一次成功应用的管理状态，以便面板失联时继续独立提供 S3 服务。
- 每份面板下发的期望状态必须具有单调递增的版本；节点必须记录已应用版本，并在重连后与面板进行版本对账。
- 直接修改节点数据库不属于受支持的常规管理方式；紧急人工修复必须能够被检测为状态漂移并重新对账。
- 批量管理必须显式选择目标节点，并分别记录每个节点的应用结果，不能把部分成功报告为全局成功。
- 控制命令必须受限于预定义操作，并具备任务标识、幂等处理、超时和结果回执；不提供任意远程 Shell。
- 日志查询必须按需执行并限制数量、时间范围和响应大小，不能把控制连接用作无界日志流。
- 存储扫描或校正属于高风险预定义任务，必须区分只读预览与实际应用，并记录操作者、目标节点、参数和结果。
- 远程升级和服务控制必须留作后续独立设计，不得在第一版协议中预留可执行任意命令的通用后门。
- 配置同步协议必须以期望状态版本为中心，允许跳过被后续版本覆盖的中间状态，并保留每次管理员变更的审计记录。
- 面板镜像与节点镜像必须能够独立构建、升级和回滚，并共享有版本约束的控制协议定义。
- 节点镜像应保留现有 S3、存储和数据面能力，但不启动现有 WebAdmin 监听服务。
- 规划必须说明现有单节点部署升级到多节点部署的兼容和迁移方式。
- 迁移失败必须保持节点原有 S3 服务和本地状态可用，并允许管理员安全重试或放弃接管。
- 面板持久状态必须可备份和恢复；恢复面板不应要求重新注册所有未被吊销且证书仍有效的节点。
- 吊销管理证书只阻止节点连接控制面，不能停止节点 S3 数据面；安全事件处置文档必须明确还需停止宿主机服务或轮换相关 S3 凭据。
- 审计日志必须至少记录操作时间、操作类型、目标节点、目标资源、结果和请求来源，且不得记录私钥、完整 S3 Secret Key、注册令牌或 Session 凭据。
- 数据库备份与 Secret Key 加密主密钥必须分别备份；仅获得数据库备份不得直接恢复出明文 Secret Key。
- 完整恢复集必须覆盖面板数据库、Secret Key 加密主密钥、根 CA 证书与加密根私钥、在线中间 CA 证书与私钥、面板配置和必要审计数据。

## Out of Scope

- S3 数据流量通过面板中转或反向隧道传输。
- 跨节点对象数据复制、纠删码或分布式存储。
- 在本规划阶段实现代码或发布镜像。

## Resolved Decisions

- 控制面管理长连接采用 **WebSocket + JSON**（HTTP Upgrade 跑在 mTLS 之上）作为协议骨架：节点主动发起 `wss://` 连接，双方在同一连接上以 `{type, version, id, payload}` 信封收发心跳、期望状态下发、任务下发和结果回执。选型理由：复用现有 `net/http` + JSON API 技术栈，仅新增一个轻量 WebSocket 库，避免引入 gRPC/protobuf 工具链；第一版节点规模小，无需 HTTP/2 级流控；JSON 可读性利于跨境调试与审计。版本协商在握手 hello 消息中交换 `protocol_version`；幂等、超时、消息数量/大小上限等背压控制按 PRD 要求手写实现。
