# 可观测性：健康检查端点 /healthz /readyz 与 Prometheus /metrics

## Goal

新增 /healthz、/readyz 健康检查端点（容器部署必备）；基于现有 DB 请求/字节统计导出 Prometheus /metrics 指标；访问日志补请求 ID 串联。

## Requirements

- Expose unauthenticated operational endpoints on the admin listener:
  - `GET /healthz` returns `200` with `text/plain` and checks only that the process can serve requests; it must not depend on the database.
  - `GET /readyz` pings the configured database; it returns `200` with `ready` when reachable and `503` with `database unavailable` when the ping fails.
  - `GET /metrics` returns Prometheus text exposition format without requiring an admin session.
- Metrics must be derived from existing persistent data only:
  - aggregate S3 request counts and byte counters from `request_stats` (`put_count`, `get_count`, `delete_count`, `bytes_in`, `bytes_out`),
  - credential count plus quota/used-byte totals from `credentials`,
  - bucket count from `buckets`,
  - database reachability as an up/down gauge.
- S3 access logs must include a generated request ID that matches the response `x-amz-request-id` used in S3 XML errors and successful S3 responses.
- The S3 request ID format must be safe for logs and HTTP headers and predictable enough to recognize as generated by this service.
- Operational endpoints must not expose credentials, secret keys, admin session state, request payloads, or object names beyond aggregate counters.
- Keep changes out of `pkg/webadmin/ui/`; this task is backend-only.

## Acceptance Criteria

- [x] `/healthz` returns HTTP `200`, `Content-Type: text/plain; charset=utf-8`, and `ok` without authentication or database access.
- [x] `/readyz` returns HTTP `200` with `ready` when `gorm.DB` can ping the underlying database and HTTP `503` with `database unavailable` when the DB handle is unavailable or closed.
- [x] `/metrics` returns HTTP `200` and `Content-Type: text/plain; version=0.0.4; charset=utf-8` in Prometheus exposition format.
- [x] `/metrics` includes `natives3_requests_total{op="put|get|delete"}`, byte totals, credential/bucket counts, quota/used bytes, and `natives3_database_up`.
- [x] Metrics query failures do not panic and are reflected by `natives3_database_up 0`.
- [x] Every S3 request response includes `x-amz-request-id`; S3 XML error bodies use the same request ID.
- [x] S3 access logs include a `request_id` field matching the response `x-amz-request-id` header, plus method, path, and elapsed duration.
- [x] Ops endpoints are registered before the SPA fallback and remain unauthenticated; existing `/api/admin/*` routes remain session-protected.
- [x] `gofmt` is applied to touched Go files, and `go build ./...` plus `go test ./...` pass or any environment limitations are reported.
