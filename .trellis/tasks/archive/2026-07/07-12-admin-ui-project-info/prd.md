# 优化管理端界面并展示项目信息

## Goal

在不改变现有管理功能和 Vue3 + Vite + ECharts 技术栈的前提下，提升管理端的视觉层级、导航辨识度和响应式体验，并让用户能直接查看项目 GitHub 地址与当前应用版本。

## Confirmed Facts

- 管理端位于 `pkg/webadmin/ui`，采用 Vue3、Vue Router、Vite 和 ECharts，并通过 Go `embed` 打包。
- 当前界面使用克制的暖灰配色、固定侧栏、简单卡片和表格，整体方向符合项目规范，但品牌信息、导航识别和侧栏底部信息较弱。
- Git 远程仓库为 `https://github.com/RSJWY/NativeS3-Bridge`。
- 前端 `package.json` 版本为 `0.1.0`，仓库最近发布标签为 `0.7.2-test`，两者语义不同。
- 发布工作流先构建前端，再构建并内嵌到 Go 二进制；版本可在前端构建阶段注入。
- 900px 以下必须切换为顶部导航，图表和表格仍需可访问且不能造成页面级横向溢出。

## Requirements

- 保留现有页面、路由、业务功能、中文文案和 API 行为。
- 优化应用外壳、品牌区、导航项、内容宽度、间距、边框、按钮状态和移动端布局的一致性。
- 在登录页和管理页提供清晰但不抢眼的 GitHub 仓库链接，并以新窗口安全打开。
- 在登录页和管理页展示当前版本号，GitHub 地址和版本信息在桌面端与移动端均可访问。
- 版本展示必须有明确且唯一的构建来源，不得将发布版本与依赖版本混淆。
- 正式 Release 和 Docker 镜像显示发布 Tag；普通本地构建回退显示 `dev`。
- 继续使用项目现有暖灰色板，不引入渐变、玻璃效果、夸张阴影、超大圆角或装饰性仪表盘元素。
- 保持键盘焦点、链接可识别性和交互控件的可访问性。

## Acceptance Criteria

- [ ] 桌面端侧栏具备更清晰的品牌、导航、GitHub 与版本信息层级。
- [ ] GitHub 链接指向 `https://github.com/RSJWY/NativeS3-Bridge`，可点击并安全地在新窗口打开。
- [ ] Release 二进制与 Docker 镜像显示对应发布 Tag，普通本地构建显示 `dev`。
- [ ] 仪表盘、密钥、桶、日志和登录页的现有功能与路由不受影响。
- [ ] 900px 以下导航和项目信息合理重排，无页面级横向滚动。
- [ ] `npm run build` 通过，Vue 类型检查和生产构建无错误。
- [ ] 使用真实浏览器检查桌面端与移动端的登录页和受保护页面布局。

## Out of Scope

- 不新增后端业务接口、数据模型或管理功能。
- 不替换 ECharts、Vue Router 或现有认证机制。
- 不进行暗色模式、多主题或整套品牌重设计。

## Decisions

- 版本号采用应用发布版本，不采用前端 `package.json` 的包版本。
- 版本通过前端构建变量注入，避免为静态项目信息增加后端接口。
