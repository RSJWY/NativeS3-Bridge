## Goal

开发一个轻量级、高性能的本地中间件，用 Go 实现，将本地操作系统的真实目录层次透明映射为标准的 Amazon S3 兼容 API。配套一个 Vue3 单密码管理界面（含 ECharts 仪表盘），用通用数据库层（本地 SQLite / 生产 MySQL / PostgreSQL）管理访问密钥、按密钥的容量配额与用量统计。最终产出**无外部运行时依赖的单文件二进制**，便于局域网跨平台一键部署。

主要使用场景：局域网内部署工具的更新分发、互动引擎（Unity/Unreal）本地资源同步、AI 自动化工作流的数据直连。

## 绝对红线（1:1 原生文件映射 — 不可妥协）

- **严禁**任何形式的数据切块（Chunking）、专有格式编码、卷合并、纠删码（Erasure Code）。
- 文件经 S3 接口上传后，必须以**原生形态**（原文件名、原后缀、原始字节内容）驻留在本地物理磁盘。
- 用户或其它单机程序必须能直接通过操作系统文件管理器查看、打开、修改这些文件。
- Bucket = 本地一级目录；Object Key（含 `/`）= 该目录下的相对路径文件。
- 允许存在的"非原生"辅助数据仅限：① 元数据 sidecar 文件；② 分段上传的临时分片目录；③ 数据库文件。它们必须明确隔离（隐藏目录或独立路径），且**绝不改变用户原始文件本身的字节与命名**。

## 技术选型（已冻结，执行者不得更改）

| 维度 | 选型 | 说明 |
|---|---|---|
| 语言 | Go (Golang) | 标准库 `net/http` + `io`，goroutine 并发，单文件二进制 |
| HTTP 框架 | Go 标准库 `net/http` | 不引入 Gin/Echo 等重框架 |
| 数据库 | GORM + 三驱动 | sqlite / mysql / postgres，同一套模型代码，配置切换 |
| 本地开发 DB | SQLite | 零部署、嵌入式 |
| 生产 DB | MySQL / PostgreSQL | 通过配置 driver + dsn 切换 |
| S3 协议响应 | `encoding/xml` | 手写标准 S3 XML 响应 |
| 签名校验 | SigV4 | 手写或借助 aws-sdk-go-v2 的 signer 校验 |
| 前端 | Vue3 + Vite | 组合式 API |
| 图表 | Apache ECharts (github.com/apache/echarts) | 仪表盘可视化 |
| 前端嵌入 | `go:embed` | 构建产物 dist/ 打入二进制，保持单文件部署 |
| 管理端鉴权 | 单密码（bcrypt + session） | 无用户名，无 admins 表 |
| 配额维度 | 按访问密钥（租户）限总容量 | `used_bytes + size > quota_bytes` 即拒绝 |

### 已否决方案（不得重新引入）
- MinIO 现代版 / Ceph / SeaweedFS：底层分块，违背红线。
- Rclone `serve s3`：黑盒进程，无法拦截事件、丢弃自定义元数据、无法对接外部鉴权。
- Rust + Tokio：纯 I/O 场景边际收益低，复杂度高，已否决。
- 多用户名管理后台：明确简化为单密码。

## 跨子任务集成验收标准（父任务负责最终验收）

- [ ] `go build` 产出**单个二进制**，无需额外运行时依赖即可在 Windows/Linux 启动。
- [ ] 同一份二进制，仅改 `config.yaml` 的 `database.driver`，即可分别用 SQLite / MySQL / PostgreSQL 正常启动并自动建表。
- [ ] 用标准 `aws-cli`（指定 `--endpoint-url`）能完成：建桶感知、上传、下载、HEAD、删除、列举、分段上传大文件。
- [ ] 上传后在本地磁盘对应目录看到**原名原后缀的原生文件**，可用系统文件管理器直接打开，内容字节一致。
- [ ] 分段上传 100MB+ 文件，完成后最终文件为单个原生文件，临时分片已清理。
- [ ] `x-amz-meta-*` 自定义元数据与 Object Tagging 上传后能被 GET/HEAD 原样取回（区别于 Rclone 丢弃元数据）。
- [ ] 未带合法签名或签名错误的请求被拒绝（返回标准 S3 403 错误 XML）。
- [ ] 某密钥已用容量达到 `quota_bytes` 后，继续上传被拒绝（标准配额超限错误）。
- [ ] 上传/删除成功后触发配置的 Webhook 事件回调，载荷含 bucket/key/size/metadata。
- [ ] 预签名 URL 可被普通 HTTP 客户端在有效期内直接 PUT/GET，过期后失效。
- [ ] 浏览器访问管理界面：单密码登录后可创建/禁用/删除密钥、设置每密钥容量配额、查看 ECharts 仪表盘（容量使用率、用量排行、请求趋势）。
- [ ] 全程未出现文件切块/专有编码（人工抽查磁盘上的文件可直接打开验证）。
